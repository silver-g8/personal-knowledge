# Personal Knowledge System (PKS v2)

> My Second Brain powered by Markdown, Git, AI, and Open Standards.

## Vision

This repository is the single source of truth for everything I learn, build, research, and maintain.

The goal is to create a lifelong knowledge system that can be understood by both humans and AI assistants.

Rather than storing isolated notes, this repository stores connected knowledge.

## Core Principles

1. Markdown First
2. Git as Source of Truth
3. Plain Text Forever
4. AI Friendly
5. Human Readable
6. Atomic Notes
7. Reusable Knowledge

## Repository Structure

```text
00-Inbox/
01-Daily Notes/
02-Learning/
03-Projects/
04-Reference/
05-Areas/
06-Resources/
07-Templates/
08-AI/
09-Archive/
```

## Folder Overview

| Folder | Purpose |
| --- | --- |
| `00-Inbox/` | Temporary capture. No permanent knowledge should stay here. |
| `01-Daily Notes/` | Daily journal, learning logs, meeting notes, and ideas. |
| `02-Learning/` | Courses, books, programming, investment, Linux, AI, and other learning material. |
| `03-Projects/` | Active projects. Each project should include `README.md`, `roadmap.md`, `tasks.md`, and `progress.md`. |
| `04-Reference/` | Stable reference materials such as cheat sheets, API docs, examples, and glossary notes. |
| `05-Areas/` | Long-term responsibilities such as health, finance, career, home, and learning. |
| `06-Resources/` | Downloaded resources such as PDFs, slides, images, and diagrams. |
| `07-Templates/` | Reusable templates for daily notes, meetings, projects, research, and book notes. |
| `08-AI/` | Prompts, agents, workflows, Codex, NotebookLM, ChatGPT, Claude, and AI operations. |
| `09-Archive/` | Completed or obsolete content. Never delete knowledge; archive it instead. |

## Workflow

```text
Capture -> Organize -> Connect -> Review -> Improve -> Reuse
```

## Naming Convention

| Item | Convention | Example |
| --- | --- | --- |
| Folders | Pascal Case, numbered when structural | `02-Learning/` |
| Files | kebab-case Markdown | `book-note-linux.md` |
| Dates | `YYYY-MM-DD` | `2026-07-04.md` |

## Git Workflow

- Use small commits.
- Write meaningful commit messages.
- Do not commit generated files unless necessary.
- Preserve history. Archive obsolete knowledge instead of deleting it.
