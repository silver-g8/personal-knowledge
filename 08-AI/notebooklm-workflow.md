# NotebookLM Workflow

Use NotebookLM for source-grounded research and synthesis.

## Suggested Workflow

1. Select a focused source set.
2. Create or update a NotebookLM notebook.
3. Ask source-grounded questions.
4. Export or summarize useful outputs.
5. Review outputs before adding them as permanent notes.

## Source Strategy

- Use one notebook for one focused topic or project.
- Prefer primary sources, user-provided notes, and durable reference material.
- Avoid mixing unrelated topics in the same notebook.
- Keep raw NotebookLM exports temporary until they are reviewed and turned into PKS notes.
- When a source has already been processed, archive the raw capture instead of deleting it.
- Record unsupported or unclear claims as open questions instead of treating them as facts.

## Repository Rule

NotebookLM output is useful, but it should not become canonical until reviewed and linked inside this repository.

## PKS Commands

Use [`learn-from-source`](commands/learn-from-source.md) when a NotebookLM source should become Learning Notes in PKS v2. The slash alias `/learn-from-source` is optional.
