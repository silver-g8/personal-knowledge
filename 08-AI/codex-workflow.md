# Codex Workflow

Use this workflow when Codex edits this repository.

## Before Editing

1. Read `AGENTS.md`.
2. Check existing files with `rg --files`.
3. Search related notes before creating new files.
4. Identify whether the task is capture, organization, editing, refactoring, or review.

## During Editing

- Preserve user intent.
- Keep edits scoped.
- Add links where useful.
- Update index files when structure changes.
- Update `CHANGELOG.md` for notable repository changes.

## After Editing

- Run a basic Markdown/file sanity check when practical.
- Summarize changed files.
- Mention any unresolved assumptions.
