# AI Review Checklist

Use this checklist before treating AI-generated content as permanent knowledge.

## Source Check

- [ ] The source material is clear.
- [ ] Important claims can be traced to a source, existing note, or user instruction.
- [ ] Unsupported claims are removed or marked as needing verification.
- [ ] The note does not invent facts, dates, links, or decisions.

## Structure Check

- [ ] The note has one clear purpose.
- [ ] The title is specific and stable.
- [ ] Headings use `##` and `###`.
- [ ] Lists and tables are readable.
- [ ] Long content is split into reusable sections.

## Repository Fit

- [ ] The note belongs in the selected folder.
- [ ] Similar notes were checked before creating a new file.
- [ ] Related notes are linked with relative links.
- [ ] Relevant indexes are updated.
- [ ] Temporary material is moved to `09-Archive/` only when it has already been processed.

## Writing Quality

- [ ] The writing is clear and beginner friendly.
- [ ] Jargon is explained or avoided.
- [ ] The note preserves the user's intent.
- [ ] AI wording was edited into natural repository style.

## Maintenance

- [ ] Frontmatter is present if the note is mature enough to need metadata.
- [ ] Tasks are actionable and not duplicated elsewhere.
- [ ] Open questions are listed clearly.
- [ ] `CHANGELOG.md` is updated when the change is notable.
